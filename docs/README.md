# 📂 โครงสร้างเอกสารโครงการ (Project Documentation)

เพื่อให้ง่ายต่อการจัดการ ได้แบ่งเอกสารออกเป็น 4 หมวดหมู่ดังนี้:

## 1. [01_สถาปัตยกรรมระบบ/](01_สถาปัตยกรรมระบบ/)
**"สถาปัตยกรรมระบบปัจจุบัน (Current Architecture)"**
หมวดนี้คือ "คัมภีร์" สำหรับ Developer ที่ต้องการเข้าใจระบบลึกซึ้ง:
*   **[🏛️ 01_สถาปัตยกรรมและโครงสร้างระบบ_ฉบับสมบูรณ์.md](01_สถาปัตยกรรมระบบ/01_สถาปัตยกรรมและโครงสร้างระบบ_ฉบับสมบูรณ์.md)**: ภาพรวม Big Picture, Data Flow, และ Tech Stack ทั้งหมด
*   **[🧠 02_เจาะลึกกระบวนการคิดของAI.md](01_สถาปัตยกรรมระบบ/02_เจาะลึกกระบวนการคิดของAI.md)**: อธิบาย Logic เบื้องหลัง Dual-Brain, Intent Filter 3 ชั้น, และ RAG Pipeline
*   **[⚙️ 02_เจาะลึกบริการหลังบ้าน.md](01_สถาปัตยกรรมระบบ/02_เจาะลึกบริการหลังบ้าน.md)**: รายละเอียด Background Workers, Redis Queue, และการเชื่อมต่อ LINE
*   **[🎭 03_เจาะลึกระบบหน้าบ้านและอวตาร.md](01_สถาปัตยกรรมระบบ/03_เจาะลึกระบบหน้าบ้านและอวตาร.md)**: การทำงานของ Frontend Director, State Management, และ Avatar Engine (Modular System)
*   **[📱 04_ระบบเชื่อมต่อ_LINE_OA_ฉบับสมบูรณ์.md](01_สถาปัตยกรรมระบบ/04_ระบบเชื่อมต่อ_LINE_OA_ฉบับสมบูรณ์.md)**: ข้อมูลจำเพาะของการเชื่อมต่อ LINE Messaging API

## 2. [02_แผนงานปัจจุบัน/](02_แผนงานปัจจุบัน/)
**"แผนงานที่กำลังพัฒนา (Active Plans)"**
เอกสาร Spec ที่ทีมกำลังดำเนินการ หรืออยู่ในคิวพัฒนา (Backlog):
*   เอกสาร Spec ที่พร้อมสำหรับการเขียนโค้ด (Implementation)
*   *ตัวอย่าง:* `แผนงานหลักโครงการ.md`, `ระบบ_Self_Correcting_RAG.md`, `แนวทางเสริมแกร่งระบบ.md`

## 3. [03_แผนอนาคต/](03_แผนอนาคต/)
**"แผนงานในอนาคต (Future Roadmap)"**
ไอเดียฟีเจอร์ที่ยังไม่อยู่ในแผนเร่งด่วน แต่เป็นวิสัยทัศน์ระยะยาว:
*   ไอเดียฟีเจอร์สร้างสรรค์ หรือการปรับเปลี่ยนสถาปัตยกรรมใหญ่ๆ ที่ยังไม่ได้เริ่มทำตอนนี้
*   *ตัวอย่าง:* `ไอเดียฟีเจอร์สร้างสรรค์.md`, `แผนอนาคต_สถาปัตยกรรมลูกผสม.md`


## 4. [04_เอกสารเสร็จแล้ว/](04_เอกสารเสร็จแล้ว/)
**"คลังเอกสารเก่า (Archived Specs)"**
เก็บเอกสารแผนงานเก่าที่ถูก Implement เสร็จสมบูรณ์แล้ว เพื่อใช้อ้างอิงย้อนหลัง (Reference)

## 5. [07_รางวัลและผลงาน/](07_รางวัลและผลงาน/)
**"หอเกียรติยศ (Hall of Fame)"**
รวบรวมรางวัล เกียรติบัตร และบทวิเคราะห์โครงการ (Post-Mortem) หลังจบการแข่งขัน:
*   **[🏆 PROJECT_SHOWCASE.md](07_รางวัลและผลงาน/PROJECT_SHOWCASE.md)**: ผลงานและความสำเร็จของโครงการ
*   **[📝 POST_MORTEM.md](07_รางวัลและผลงาน/POST_MORTEM.md)**: บทเรียนและสิ่งที่เรียนรู้จากการทำงาน
